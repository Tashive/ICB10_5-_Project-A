# 국민연금 가입 사업장 내역 HTML 정적 대시보드 구축 프롬프트

본 문서는 `nps-company/report` 폴더의 데이터 탐색(EDA) 결과와 기존 분석 시각화 자료를 바탕으로, 국민연금 가입 사업장 현황을 보여주는 **HTML 기반의 정적 웹페이지 대시보드**를 구축하기 위한 계획 및 지시 프롬프트입니다. AI 에이전트 또는 개발자는 아래 지시사항에 따라 대시보드를 구현하십시오.

## 1. 프로젝트 개요
- **목적**: 2026년 3월 기준 58만여 개의 국민연금 가입 사업장 데이터 분석 결과를 웹 브라우저에서 서버 없이 바로 확인할 수 있는 정적 HTML 대시보드 개발
- **기반 데이터 요약 (EDA 리포트 참고)**:
  - **전체 행 수**: 582,825건
  - **수치형 특징**: 가입자수(평균 19.5명, 중앙값 6명) 및 당월고지금액(평균 715만원, 중앙값 191만원)의 극심한 양극화 (롱테일 분포).
  - **범주형 특징**: 등록 상태 94.2%, 법인 사업장 90.1% 차지. 지역별로는 경기도(41), 서울(11) 중심으로 강한 수도권 집중화 현상.
  - **주요 업종 키워드**: BIZ_NO미존재사업장, 제조업, 도매업, 공사업 순으로 다수 분포.

## 2. 대시보드 주요 기능 및 구성 요소

정적 웹 환경의 특성상 58만 건의 원본 데이터를 직접 로딩하는 대신, 이미 산출된 **요약 통계 데이터(CSV/JSON)** 및 **기존 시각화 이미지(`nps-company/images/`)**를 적극 활용하여 대시보드를 구성합니다.

### A. 요약 지표 패널 (Key Metrics)
대시보드 상단에 주요 통계 수치를 강조하는 카드 형태의 UI를 배치합니다.
- 총 사업장 수
- 가입자 수 평균 및 중앙값
- 당월고지금액 평균 및 중앙값
- 신규 취득자 수 / 상실 가입자 수 평균

### B. 주요 시각화 영역 (Main Dashboard)
다음 항목들을 적절한 레이아웃(예: CSS Grid 또는 Flexbox)을 사용하여 배치합니다. 자바스크립트 차트 라이브러리를 사용하거나 기존 생성된 이미지를 활용합니다.

1. **가입자수 및 고지금액 분포**
   - 롱테일 분포를 보여주는 로그 스케일 차트 혹은 관련 이미지(`vis_01_subscriber_dist.png`, `vis_02_payment_dist.png` 등) 배치.
2. **지역별/업종별 현황**
   - 지역별 밀집도 및 상위 업종 현황을 보여주는 바 차트 또는 파이 차트 구현 (Chart.js 등 활용) 또는 기존 이미지 배치.
3. **사업장 상태 및 형태 비율**
   - 등록/탈퇴, 법인/개인 비율을 도넛 차트로 구현하여 시각적 포인트 제공.
4. **인사이트 텍스트 요약**
   - 각 차트 또는 섹션 하단에 EDA 리포트에서 도출된 주요 결론(양극화, 수도권 집중 등)을 텍스트로 요약하여 제공.

## 3. 기술 스택 및 구현 요구사항
- **언어**: HTML5, CSS3, Vanilla JavaScript
- **프레임워크/라이브러리**:
  - 스타일링: Tailwind CSS (CDN 방식) 또는 Vanilla CSS를 사용하여 모던하고 깔끔한 UI 구성
  - 데이터 시각화: Chart.js, ECharts, 또는 Plotly.js (CDN 방식 로드) 또는 `<img>` 태그를 활용한 정적 이미지 매핑
- **데이터 로딩**: `nps-company/report` 경로의 `cat_desc.csv`, `num_desc.csv` 등 요약된 데이터를 JavaScript의 `fetch()` API로 불러와 파싱하여 차트/지표에 연동하거나, 데이터가 하드코딩된 형태로 구성.

## 4. 실행 및 개발 지침
1. `index.html` 파일을 메인으로 하여 대시보드 뼈대를 작성합니다.
2. CSS를 활용해 반응형 웹 디자인(모바일 및 데스크톱 뷰 지원)을 적용합니다.
3. 프로젝트 내 파일 경로 및 이미지 경로는 반드시 **상대경로**(예: `../images/vis_01_subscriber_dist.png`)를 사용하십시오.
4. 외부 라이브러리는 별도의 설치(npm 등) 없이 CDN 링크를 `<head>` 태그에 포함시켜 사용합니다.
5. 복잡한 백엔드 로직 없이 웹 브라우저에서 파일을 열었을 때(`file://` 프로토콜 또는 로컬 라이브 서버) 즉시 화면이 렌더링되도록 보장하십시오.
